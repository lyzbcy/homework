# 上传到 GitHub 的操作步骤

由于路径编码问题，请按照以下步骤手动操作：

## 方法一：使用 Git 命令（推荐）

1. **创建目录结构并移动文件**
   ```bash
   # 创建目标目录
   mkdir -p Knowledge-Engineering/QAMedicalKG
   
   # 移动所有文件（排除 .git 和当前脚本）
   git mv * Knowledge-Engineering/QAMedicalKG/ 2>/dev/null || true
   git mv .* Knowledge-Engineering/QAMedicalKG/ 2>/dev/null || true
   
   # 排除不需要移动的文件
   git reset HEAD Knowledge-Engineering
   git checkout -- Knowledge-Engineering
   ```

2. **提交更改**
   ```bash
   git add -A
   git commit -m "Reorganize files to Knowledge-Engineering/QAMedicalKG/"
   ```

3. **推送到远程仓库**
   ```bash
   git branch -M main
   git push -u origin main
   ```

## 方法二：使用 GitHub Desktop 或 Web 界面

1. 在本地创建 `Knowledge-Engineering/QAMedicalKG/` 目录
2. 将所有文件移动到该目录
3. 使用 GitHub Desktop 提交并推送

## 方法三：直接推送当前内容

如果上述方法都有问题，可以先推送当前内容，然后在 GitHub Web 界面上使用文件移动功能重新组织。
